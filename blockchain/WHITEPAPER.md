# OWNIA 백서 (Whitepaper)

---

## 1. 개요 (Abstract)

OWNIA는 **데이터 주권 + AI 기반 3D 생성 + 도시 탐색 경제**를 결합한 도시형 소셜 플랫폼이다. 사용자가 생성한 데이터의 소유권을 보장하고, 텍스트에서 3D 객체를 생성하는 AI 마법을 통해 디지털 자산을 만들며, 도시 탐색 활동에 경제적 보상을 부여한다.

OWNIA 토큰은 이 생태계의 경제적 기반으로서, Solana 블록체인 위에 구축된 SPL 토큰이다.

---

## 2. 비전 (Vision)

> *"데이터 주권 + AI 마법 + 항해적 경제"*

| 핵심 축 | 설명 |
|---------|------|
| **데이터 주권** | 사용자가 자신의 데이터를 소유하고 거래할 권리 |
| **AI 마법** | 텍스트 → 3D 객체 생성, AI 기반 콘텐츠 창작 |
| **항해적 경제** | 도시 탐색·발견 활동에 대한 토큰 보상 체계 |

2026년 트렌드인 **AI + 블록체인 + 메타버스**의 교차점에서, OWNIA는 사용자가 단순 소비자가 아닌 **데이터의 주인이자 창작자**가 되는 플랫폼을 지향한다.

---

## 3. 문제 정의 (Problem Statement)

1. **데이터 착취**: 빅테크가 사용자 데이터를 무상 수집하여 수익 창출
2. **창작 장벽**: 3D 콘텐츠 제작에 전문 기술과 고가 툴 필요
3. **도시 경험의 단절**: 오프라인 활동이 디지털 경제와 연결되지 않음
4. **불투명한 보상**: 크리에이터 보상 체계의 불투명성과 중앙화

---

## 4. 솔루션 (Solution)

### 4.1 데이터 소유권 레이어
- 사용자 생성 데이터에 대한 온체인 소유권 증명
- 데이터 거래 시 창작자에게 직접 보상
- 개인정보 보호와 투명성의 균형

### 4.2 Text-to-3D AI 엔진
- 텍스트 프롬프트로 3D 객체 자동 생성
- 생성된 3D 자산의 NFT화 및 거래
- AI 서비스 이용 시 OWNIA 토큰으로 결제

### 4.3 도시 탐색 경제 (Navigation Economy)
- 실제 도시 탐색 활동에 토큰 보상
- 지역 상권과 연결된 리워드 시스템
- 커뮤니티 기반 도시 정보 크라우드소싱

## 6. 기술 아키텍처 (Technical Architecture)

### 6.1 왜 Solana인가?

| 특성 | Solana | Ethereum | OWNIA에 대한 이점 |
|------|--------|----------|-------------------|
| TPS | ~65,000 | ~15 | 실시간 보상 처리 |
| 수수료 | ~$0.00025 | ~$1-50 | 마이크로 트랜잭션 가능 |
| 확인 시간 | ~400ms | ~12s | 즉각적 UX |
| NFT 생태계 | 강력 | 강력 | 3D 자산 NFT화 |

### 6.2 스마트 컨트랙트 구조

Anchor 프레임워크 기반, 10개 온체인 명령어:

```
ownia-token (Anchor Program)
├── initialize            — 토큰 민트 + 프로그램 설정 초기화
├── mint_tokens           — 관리자 전용 토큰 발행 (공급 상한 검증)
├── create_vesting        — 베스팅 스케줄 생성 + 토큰 잠금
├── claim_vesting         — cliff 경과 후 선형 언락 수령
├── initialize_stake_pool — 스테이킹 풀 생성
├── stake                 — 토큰 예치 (추가 스테이킹 가능)
├── unstake               — 잠금 해제 후 토큰 + 보상 회수
├── claim_rewards         — 스테이킹 보상 수령
├── create_proposal       — 거버넌스 제안 생성 (최소 10,000 OWNIA)
└── cast_vote             — 스테이킹 가중 투표 (이중 투표 방지)
```

### 6.3 온체인 계정 구조

| 계정 | PDA Seed | 역할 |
|------|----------|------|
| `ProgramConfig` | `["config"]` | 전역 설정, 관리자, 민트 주소, 총 발행량 |
| `VestingSchedule` | `["vesting", beneficiary, id]` | 수혜자별 베스팅 정보 |
| `StakePool` | `["stake_pool"]` | 풀 설정, 보상률, 총 스테이킹 |
| `StakeAccount` | `["stake", user]` | 사용자별 스테이킹 정보 |
| `Proposal` | `["proposal", id]` | 거버넌스 제안 내용 및 투표 집계 |
| `VoteRecord` | `["vote", proposal, voter]` | 이중 투표 방지 기록 |

### 6.4 보상 계산 공식

```
연간 보상 = staked_amount × reward_rate_bps / 10,000

실시간 보상 = staked_amount × reward_rate_bps × elapsed_seconds
             ÷ (10,000 × 31,536,000)
```

기본 보상률: **1,000 BPS (연 10%)**

## 9. 보안 (Security)

- **Anchor 프레임워크**: 계정 유효성 자동 검증
- **PDA 기반 권한**: 프로그램 서명으로 중앙 키 위험 제거
- **공급 상한 하드코딩**: 10억 초과 발행 불가
- **관리자 권한 분리**: 민팅/베스팅은 관리자 전용
- **이중 실행 방지**: PDA init으로 중복 트랜잭션 차단
- **오버플로우 방지**: checked math 적용
- **향후 계획**: 외부 보안 감사 (Mainnet 배포 전)
